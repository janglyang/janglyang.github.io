使用GitHub+Hexo搭建博客的笔记

年纪大了，不想太折腾，再次搭建博客的想法仅仅是做一些简单的记录，所以选择了GitHub pages，同理选择了十分流行的Hexo。

在本地安装Git，node.js和Hexo。前两项均可以去官网下载安装包进行安装，node.js安装完成后会自动写入环境变量，Git需要手动设置环境变量（两个），分别将bin和git-core所在的安装路径添加进环境变量。然后通过命令行安装Hexo。

新建文件夹，比如/blog，命令行进入该文件夹。

将本地的/blog发布到远端的GitHub库中。在GitHub中新建库，名称为用户名.github.io。需要知道此库的http地址，个人的GitHub用户名和密码。

修改_config.yml文件

然后进行第一次发布。

按照提示输入GitHub用户名和密码。

主题的安装。一般某一具体主题的主页会提供该主题的安装和使用说明。

现在我的博客写作发布操作为：

  1. Hexo n “标题”；
  2. 使用文本编辑器notepad++写作markdown；
  3. Hexo g 生成博客；
  4. Hexo d 发布到远端。

平时随时写作，回到台式机时统一使用上述流程发布，某篇博客的写作发布时间是可以在文本编辑器中更改的，所以是可以如实反映我的写作时间先后的。
